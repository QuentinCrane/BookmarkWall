# Release Notes

本文件记录 BookmarkWall 的主要用户可见变化。版本号以 `manifest.json` 和 `package.json` 为准。

## v0.7.10

- 当前稳定版本号更新为 `0.7.10`。
- 继续完善首页海报布局、设置页操作区、移动端适配和中英文界面文本。
- 保持首次授权门槛：用户确认前不自动访问网页、不启动真实截图队列。
- 保持截图队列停止清理逻辑，停止后会关闭已追踪的临时截图标签页和窗口。

## v0.7.9

- 修复未授权截图时首页出现大量堆叠线框占位导致卡顿的问题。
- 新增轻量本地预览封面，不访问外部 favicon，不访问书签网页。
- 统一首页卡片尺寸：固定封面区与底部文字区，避免底部过高或卡片挤压。
- 未获得用户同意时，首页只显示“查看引导并授权”，不会启动并发截图。
- 缩略图缓存 schema 升级为 `poster-local-design-v7`。

## v0.7.7

- 首次打开不再自动开始网页截图，必须用户在引导页勾选同意并点击“开始生成真实海报”后才会开始。
- 首页卡片重设为紧凑固定几何尺寸，封面区固定 16:9，文字区固定高度。
- 首次引导按三栏结构重做：欢迎说明、安全备份、截图配置与明确授权说明。
- 品牌图标新增本地书签图标，首页侧栏视觉更接近参考设计。
- 重构本地兜底海报，避免真实截图失败时出现条状、空白或 favicon 大图。
- 公开封面无法下载、为空白、过细或低信息量时，统一回退为本地设计海报。
- 不再将 favicon-only 结果作为最终海报缓存。

## v0.7.2-hotfix

- 新增截图质量检测，过白、信息量过低或细条状截图不会写入真实截图缓存。
- 新增本地设计海报兜底，网站无法正常渲染时仍能显示稳定 16:9 封面。
- 缓存 schema 升级至 `poster-crop-v3`，安装后会清理旧版条状截图缓存并重新生成。
- 保留大范围生成逻辑，当前可见书签会尽量全部处理，不再只处理 24 个。

## v0.7.1-hotfix

- 修复智能裁剪在白底网页上误把横向文字线条识别为主体区域的问题。
- 增加裁剪安全兜底，检测区域过窄、过扁或高度不足时回退到首屏 16:9 裁剪。
- 自动迁移旧版本缩略图缓存，清除此前保存的异常截图和失败记录。
- 取消旧逻辑里的 24 张自动生成硬上限。

## v0.7.0

- 新增书签安全备份闭环，降低移动、删除或 AI 批量移动前的误操作风险。
- 首次欢迎向导新增“先导出全部书签”安全步骤。
- 支持一键导出全部浏览器书签为 HTML 文件，可通过 Chrome / Edge 书签管理器重新导入。
- 支持导出全部书签 JSON 备份，便于审查完整书签树和排查问题。
- 设置页“数据与缓存”新增书签安全备份专区。
- 新增“整理前如果未备份则提醒”开关。
- 主海报墙、设置页、首次引导和 AI 预整理结果页完成一轮视觉重构。

## v0.6.0

- 默认启用增强 CDP 真实截图模式。
- 新增 `chrome.debugger` + Chrome DevTools Protocol 截图引擎。
- 新增 CDP 失败后自动回退临时窗口截图。
- 新增临时窗口失败后自动回退网页公开封面 / favicon。
- 新增截图进度中的 CDP / 窗口 / 降级 / 失败统计。
- 新增首次打开欢迎向导，说明真实网页截图海报、整理流程、隐私和 AI 默认关闭策略。
- 截图并发数支持 1-63，并在设置页以滑杆形式实时调整。
- 新增本地 SVG 图标系统，替换界面中杂乱的 Unicode 符号。
- 更新 UI 和 Manifest 测试，覆盖并发范围、首次引导、图标系统和海报化管线。

## v0.3.0

- 新增临时窗口真实截图队列。
- 支持自动截图、失败降级和截图进度提示。

## v0.2.0

- 新增自动公开封面图生成。
- 支持 `og:image` / `twitter:image` / favicon 兜底。

## v0.1.x

- 建立 MV3 扩展结构。
- 实现书签读取、海报墙、拖拽、多选、AI 配置和设置页。
