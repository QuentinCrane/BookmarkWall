# 测试说明

## 自动测试

```bash
npm run check
npm test
```

覆盖内容：

- URL 归一化。
- 重复书签检测。
- 书签树扁平化。
- 文件夹计数。
- 本地 AI 推荐。
- 网页封面 meta 提取。
- Manifest V3 和权限检查。

## 手动验收清单

### 书签基础

- [ ] 加载后能显示真实浏览器书签数量。
- [ ] 左侧文件夹树能展开。
- [ ] 点击文件夹后海报墙切换内容。
- [ ] 搜索、排序、筛选可用。
- [ ] 多选、移动、删除、撤销可用。

### 真实截图

- [ ] 默认进入文件夹后自动开始生成真实截图。
- [ ] CDP 增强截图成功时，卡片显示真实网页首屏。
- [ ] 默认 CDP 失败时不会自动弹临时窗口，会降级为封面并在卡片标记失败。
- [ ] 手动选择临时窗口模式时，才使用临时窗口补拍。
- [ ] 临时窗口失败时降级到网页封面 / 本地海报。
- [ ] 设置页能切换截图方式。
- [ ] 清除缓存后能重新生成。

### AI

- [ ] AI 默认关闭。
- [ ] 设置页能配置 Provider。
- [ ] 开启 AI 后主界面显示 AI 入口。
- [ ] AI 结果必须预览确认后才能应用。

## 容器限制

当前运行环境的 Chromium 被系统策略限制，无法完成自动浏览器截图验收。请在本机 Chrome / Edge 中加载扩展进行真实截图验证。

### v0.7.x 海报与授权验收

- [ ] 设置页显示“海报截图设置”，而不是旧的“缩略图设置”。
- [ ] 可设置并发截图数量 1 / 4 / 16 / 63，并且滑杆与数字输入保持同步。
- [ ] 可设置截图视口、裁剪模式、白底增强。
- [ ] 真实截图完成后卡片顶部没有遮挡画面的“真实截图”大浮标。
- [ ] 白底网页生成的海报不再像空白纸，应有主体叠层或更清晰的裁剪区域。
- [ ] 当前可见卡片优先被截图，海报墙逐步从占位图变成真实网页海报。
- [ ] 未完成截图授权时，首页不会自动访问网页或启动截图队列。
- [ ] 点击停止后，正在运行的截图标签页或临时窗口会被关闭。
