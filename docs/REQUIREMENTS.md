# 需求文档 v0.6.0

## 产品定位

书签海报墙是一款 Chrome / Edge 浏览器扩展。它不是新标签页导航，也不是云收藏夹，而是一个独立扩展页面，用“海报墙 + 真实网页截图”的方式帮助用户整理浏览器原生书签。

核心目标：

> 让用户通过真实网页画面识别书签内容，并像整理海报墙一样完成移动、分类、删除、合并和 AI 预整理。

## 主界面

- 左侧：文件夹树和特殊入口。
- 中间：海报墙主视图。
- 顶部：搜索、范围、排序、筛选、卡片大小、AI 入口。
- 底部：多选后出现批量操作栏。

## 核心功能

### 1. 原生书签管理

- 读取原生书签树。
- 新建文件夹。
- 移动书签。
- 编辑标题和 URL。
- 删除并撤销。
- 多选批量移动。
- 重复书签检测。

### 2. 真实网页截图海报

真实截图是核心功能。插件默认采用多级海报生成引擎：

1. 增强 CDP 真实截图。
2. 临时窗口真实截图。
3. 当前标签页截图。
4. `og:image` / `twitter:image` 降级。
5. favicon / 模拟网页占位。

要求：

- 进入文件夹后自动排队生成缺失海报。
- 截图保存本地缓存。
- 支持清除缓存和重新生成。
- 支持停止队列。
- 失败时自动降级，不出现空白卡片。

### 3. 占位海报

没有真实截图时，不使用普通渐变色块。占位海报应模拟网页缩略图结构，包括：

- 浏览器顶部栏。
- favicon。
- 域名。
- 页面标题。
- 模拟内容线条。
- “等待真实截图”提示。

### 4. AI 预整理

AI 是可选增强功能。

- 默认关闭。
- 在设置中配置 API 和模型。
- 支持 OpenAI-Compatible API。
- 用户主动触发后才发送数据。
- AI 只生成建议。
- 所有建议必须预览确认后才能应用。
- 支持批量应用高置信度建议。

### 5. 权限与隐私

- 本地优先。
- 不上传书签和截图到开发者服务器。
- AI 默认关闭。
- 截图缓存可清除。
- `debugger` 权限仅用于生成真实网页海报。

## 成功标准

- 打开扩展后能读取真实书签。
- 当前文件夹能显示海报墙。
- 缺失海报能自动生成真实网页截图。
- 无真实截图时占位视觉接近网页预览，而不是普通渐变图。
- 主界面接近概念设计图。
- 整理功能可用。
- AI 功能不打扰主体验。


## v0.7.0 安全备份补充

- 第一次进入必须提供快速导出全部书签功能。
- 导出 HTML 备份应优先展示，因为它可以重新导入浏览器书签管理器。
- 导出 JSON 备份作为完整书签树审查和开发排查用途。
- 在移动、删除、重命名或应用 AI 批量整理前，如用户尚未备份，应默认提醒先导出。
